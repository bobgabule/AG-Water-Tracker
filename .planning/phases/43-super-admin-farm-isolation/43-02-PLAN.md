**Part A: Fix the null-to-empty-string conversion bug in WellAllocationsPage.tsx.**

Find the current code near the top of the component:
```typescript
const { farmId: activeFarmId, farmName: activeFarmName } = useActiveFarm();
const farmName = activeFarmName ?? '';
const farmId = activeFarmId ?? '';  // BUG: '' is truthy in guards
```

Replace with:
```typescript
const { farmId: activeFarmId, farmName: activeFarmName } = useActiveFarm();
const farmName = activeFarmName ?? '';
```

Then remove the `farmId` alias entirely and use `activeFarmId` directly. Find all uses of `farmId` in the file and replace with `activeFarmId`.

Add an early return guard after the hooks section (before any rendering logic, but after all hooks have been called — React rules of hooks require all hooks to run unconditionally). Place it after the very last hook in the component, right before the first return statement that renders UI:

```tsx
if (!activeFarmId) {
  return (
    <div className="min-h-screen bg-surface-page flex items-center justify-center">
      <p className="text-white/60">No farm selected</p>
    </div>
  );
}
```

IMPORTANT: The early return MUST come AFTER all hooks (useState, useCallback, useEffect, useMemo, etc.) are called. React forbids conditional hook calls. So place the guard after the very last hook in the component.

If that is not feasible (hooks are interspersed with logic that references farmId), instead keep `const farmId = activeFarmId;` (no fallback) and add a null guard:
```typescript
const farmId = activeFarmId;
```
Then add `if (!farmId) return <early return UI>;` after ALL hooks.

This ensures null farmId is not silently converted to empty string, which would pass through `if (!farmId)` guards as truthy.
