Modify FarmSelector.tsx with three changes:

1. **Auto-select first farm for super_admin (with hydration guard)**: Add a `useEffect` after the `options` memo. The effect MUST wait for Zustand persist hydration before running — otherwise it would see `overrideFarmId === null` on page reload before localStorage has been read, and overwrite the user's previously persisted selection.

First, track hydration state. Use `useState` + `useEffect` to subscribe to hydration:
```tsx
const [hydrated, setHydrated] = useState(useActiveFarmStore.persist.hasHydrated());
useEffect(() => {
  const unsub = useActiveFarmStore.persist.onFinishHydration(() => setHydrated(true));
  return unsub;
}, []);
```

Then, the auto-select effect:
```tsx
useEffect(() => {
  if (!hydrated) return; // Wait for persist rehydration
  if (role !== 'super_admin') return;
  if (options.length === 0) return;

  const overrideId = useActiveFarmStore.getState().overrideFarmId;

  // Deleted-farm fallback: stored farm no longer exists
  if (overrideId && !options.find(o => o.id === overrideId)) {
    useActiveFarmStore.getState().clearOverride();
    // clearOverride sets overrideFarmId to null, which will re-trigger this effect
    return;
  }

  // Auto-select first farm when no override is set
  if (!overrideId) {
    const { setActiveFarm } = useActiveFarmStore.getState();
    setActiveFarm(options[0].id, options[0].name);
  }
}, [hydrated, role, options]);
```

Note: Reading from `useActiveFarmStore.getState()` inside the effect (instead of subscribing to `overrideFarmId` as a dependency) is intentional — we only want this logic to run when `hydrated`, `role`, or `options` change, not on every override change (which would cause loops).
